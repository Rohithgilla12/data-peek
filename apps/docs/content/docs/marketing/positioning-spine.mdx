# Positioning Spine

Use this as the default messaging source for Reddit posts, dev.to articles, launch blurbs, and comments.

## Core Positioning Statement

data-peek is a fast desktop SQL client for developers who need answers quickly, with AI assistance and performance tooling, without enterprise tool bloat.

## One-Liner Variants

- Fast desktop SQL client for developers, with AI and deep query diagnostics.
- Query your data in seconds, profile bottlenecks, and ship fixes faster.
- A minimal SQL client that feels local-first, keyboard-first, and developer-first.

## Audience

- Backend and full-stack developers debugging production or staging data.
- Builders who run many short query sessions per day and value startup speed.
- Individual developers and small teams that do not need heavyweight admin suites.

## Messaging Pillars

## 1) Speed And Focus

- Opens in under 2 seconds.
- Built for quick "peek and verify" workflows.
- Keyboard-first interactions and command palette.

## 2) AI With Boundaries

- Natural language to SQL and chart generation with BYOK.
- Schema-aware context improves query quality.
- Schema metadata only, not row-level data.

## 3) Performance Visibility

- Query telemetry breaks down query lifecycle by phase.
- Benchmark mode provides percentile metrics (P90/P95/P99).
- EXPLAIN and query plan workflows support optimization loops.

## 4) Local-First Trust

- Credentials protected with OS keychain encryption.
- No app telemetry statement for user trust.
- Strong fit for privacy-sensitive developer workflows.
