# Reusable Post Templates

Use these templates as starting points. Replace bracketed placeholders and keep language direct.

## Template 1: Launch / "I Built This" (Reddit)

**Title format examples**

- `[data-peek] - Fast SQL client for developers with AI and query telemetry`
- `I built a fast SQL client because I was tired of heavyweight database tools`

**Body**

I built **data-peek** because [specific pain you had].

What I wanted:
- [Outcome 1]
- [Outcome 2]
- [Outcome 3]

What it currently does:
- [Feature 1]
- [Feature 2]
- [Feature 3]

What surprised me while building:
1. [Unexpected lesson]
2. [Tradeoff]
3. [What you would change]

I am the builder, and I would love feedback on **[one specific question]**.
If useful, project link is here: [link]

## Template 2: Technical Deep-Dive (dev.to or strict subreddit)

**Title formula**

`How We [Built X] in [Stack] and Reduced [Pain/Latency/Complexity]`

**Outline**

1. Context: what problem we observed in real usage.
2. Constraints: speed, safety, compatibility, developer experience.
3. Implementation:
   - Architecture overview
   - Core data flow
   - Tradeoffs rejected
4. Results:
   - What improved
   - What still breaks
5. Lessons:
   - One thing to copy
   - One thing to avoid

**Closing CTA**

"If you have built something similar, what would you change first in this approach?"

## Template 3: Weekly Retrospective / Build-in-Public

**Title formula**

`Week [N] Building data-peek: [One win], [One failure], [Next move]`

**Body**

This week shipping summary:
- Shipped: [Feature or improvement]
- Broke: [Issue]
- Learned: [Behavioral insight from users]

Distribution summary:
- Posted in: [Subreddit + article]
- Best response: [What resonated]
- Worst response: [What was ignored]

Metrics snapshot:
- Upvotes/comments: [numbers]
- Clicks/stars/installs: [numbers]
- Conversion insight: [observation]

Next week:
1. [Priority 1]
2. [Priority 2]
3. [Priority 3]

Question for builders: **[specific question]**

## Template 4: Comment Reply Playbook (For first 4 hours)

Use these fast responses to keep engagement high and authentic.

- **Feature request:** "Good call. We prioritized [X] first because [reason], but [request] is on the shortlist."
- **Skepticism:** "Fair concern. Here is what we measured/observed: [data]."
- **Comparison question:** "Compared to [tool], our focus is [narrow use case], not [broad use case]."
- **Security concern:** "Reasonable ask. We currently do [control], and we do not do [risky behavior]."
- **Pricing concern:** "Pricing is [model]. I am optimizing for [fairness goal], not [maximize extraction]."

## Red Flags Checklist Before Publishing

- No clickbait claims.
- No "DM me" CTA.
- No cross-posting identical copy.
- One clear feedback request included.
- Affiliation disclosed in first paragraph.
