Connection Health Monitor — What should it show?

Pick the metrics that matter most (select multiple)

1

Active Queries

List of currently running queries with PID, duration, user, state, and query text. Ability to kill long-running queries. Like pg_stat_activity but pretty.

2

Database & Table Sizes

Total database size, per-table sizes, index sizes, bloat estimates. Sortable table with visual size bars.

3

Connection Pool Stats

Active/idle/waiting connections, max connections, connections by user/application. Gauge charts showing utilization.

4

Replication Status

Replica lag, replication slots, WAL position. Important for production monitoring but niche.

5

Cache Hit Ratios & Performance

Buffer cache hit ratio, index usage stats, sequential vs index scans. Helps identify performance tuning opportunities.

6

Locks & Blocking

Current locks, blocked queries, lock wait times. Critical for debugging production deadlocks and contention.
