CSV Import — How smart should the mapping be?

When importing a CSV, how much should the app figure out on its own?

A

Simple — Manual Mapping

User picks target table, then maps each CSV column to a table column via dropdowns. Type mismatches shown as warnings. Reliable, predictable, no magic.

  • File picker or drag-and-drop
  • Preview first 10 rows
  • Manual column-to-column mapping
  • Basic type validation
  • Import with progress bar
B

Smart — Auto-Map + Create Table

Auto-detect column names and types from CSV headers. Auto-map to matching table columns. Option to create a new table from the CSV structure. User can override everything.

  • Everything in Simple, plus:
  • Auto-match CSV headers → table columns by name
  • Infer types from data (int, float, date, text)
  • "Create new table" option with inferred schema
  • Preview generated INSERT/COPY SQL
C

Full — Smart + Batch Options

Everything in Smart, plus options for large files: batch size, conflict handling (ON CONFLICT), truncate before import, and transaction wrapping.

  • Everything in Smart, plus:
  • Configurable batch size (100/500/1000 rows)
  • ON CONFLICT strategy (skip/update/error)
  • Truncate table before import option
  • Transaction wrapping (all-or-nothing)
  • COPY command for PostgreSQL (fastest)
